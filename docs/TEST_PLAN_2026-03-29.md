## 1. Overview

This test plan covers the 5 new features implemented in dime_base v1.1:

| Feature | Description |
|---------|-------------|
| **FR-DimeCfg** | Dime Configuration (LLM backend, tone, mode, privacy) |

## 3. Test Cases

### 3.1 FR-DimeCfg - Dime Configuration

| Test ID | Test Case | Expected Result |
|---------|-----------|-----------------|
| CFG-01 | Get default config | Returns `llmBackend: 'deepseek'`, `tone: 'casual'`, `mode: 'assistant'` |
| CFG-02 | Update config | Config updated, `success: true` |
| CFG-03 | API key masking | `apiKey` returned as `'***'` not actual value |
| CFG-04 | Reset config to defaults | Config reset to factory defaults |
| CFG-05 | Invalid LLM backend | Returns 400 error with `INVALID_CONFIG` |
| CFG-06 | Custom without endpoint | Returns 400 error |
| CFG-07 | Retention out of range (>365) | Returns 400 error |

#### Test Commands

```bash
# CFG-01: Get default config
curl http://localhost:3000/api/config/DIME_ID \
  -H "x-owner-id: OWNER_ID"

# CFG-02: Update config
curl -X PUT http://localhost:3000/api/config/DIME_ID \
  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
  -d '{
    "llmBackend": "openai",
    "tone": "formal",
    "mode": "analytical",
    "privacy": {
      "dataSharing": true,
      "conversationRetention": 60
    }
  }'

# CFG-03: API key masking
curl -X PUT http://localhost:3000/api/config/DIME_ID \
  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
  -d '{"llmBackend": "custom", "customEndpoint": "https://api.example.com", "apiKey": "secret123"}'

# CFG-04: Reset to defaults
curl -X POST http://localhost:3000/api/config/DIME_ID/reset \
  -H "x-owner-id: OWNER_ID"

# CFG-05: Invalid backend
curl -X PUT http://localhost:3000/api/config/DIME_ID \
  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
  -d '{"llmBackend": "invalid"}'

# CFG-06: Custom without endpoint
curl -X PUT http://localhost:3000/api/config/DIME_ID \
  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
  -d '{"llmBackend": "custom"}'

# CFG-07: Retention out of range
curl -X PUT http://localhost:3000/api/config/DIME_ID \
  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
  -d '{"privacy": {"conversationRetention": 500}}'
```

### 3.2 FR-D2D - Dime-to-Dime Communication

| Test ID | Test Case | Expected Result |
|---------|-----------|-----------------|
| D2D-01 | Create channel | Channel created with `status: 'active'` |
| D2D-02 | Get channel info | Returns channel with participants |
| D2D-03 | Send message | Message sent, ID returned |
| D2D-04 | Get message history | Messages returned in chronological order |
| D2D-05 | Privacy - owner cannot access | Owner not in participants, access denied |
| D2D-06 | WebSocket events | Events emitted to channel participants |
| D2D-07 | Close channel | Channel status changed to 'closed' |

#### Test Commands

```bash
# D2D-01: Create channel (requires two dimes from different owners)
curl -X POST http://localhost:3000/api/d2d/channels \
  -H "Content-Type: application/json" \
  -d '{"dimeA": "DIME_A_ID", "dimeB": "DIME_B_ID"}'

# D2D-02: Get channel info
curl http://localhost:3000/api/d2d/channels/CHANNEL_ID

# D2D-03: Send message
curl -X POST http://localhost:3000/api/d2d/channels/CHANNEL_ID/messages \
  -H "Content-Type: application/json" \
  -d '{"fromDimeId": "DIME_A_ID", "content": "Hello from Dime A!"}'

# D2D-04: Get message history
curl http://localhost:3000/api/d2d/channels/CHANNEL_ID

# D2D-07: Close channel
curl -X DELETE http://localhost:3000/api/d2d/channels/CHANNEL_ID \
  -H "Content-Type: application/json" \
  -d '{"requesterDimeId": "DIME_A_ID"}'
```

#### WebSocket Test (JavaScript)

```javascript
const socket = io('http://localhost:3000');

socket.emit('join_dime', { dimeId: 'DIME_A_ID' });

socket.emit('d2d_create_channel', { dimeA: 'DIME_A_ID', dimeB: 'DIME_B_ID' });
socket.on('d2d_channel_created', (data) => console.log('Channel created:', data));

socket.emit('d2d_send_message', { channelId: 'CHANNEL_ID', fromDimeId: 'DIME_A_ID', content: 'Hello!' });
socket.on('d2d_message', (data) => console.log('New message:', data));
```

### 3.3 FR-Admin - Admin Dashboard

| Test ID | Test Case | Expected Result |
|---------|-----------|-----------------|
| ADM-01 | Get system stats | Returns `{activeUsers, totalDimes, activeDimes, ...}` |
| ADM-02 | List playgrounds | Returns array of playgrounds |
| ADM-03 | Create playground | Playground created, ID returned |
| ADM-04 | Update playground | Playground settings updated |
| ADM-05 | Delete playground | Playground deleted |
| ADM-06 | List owners | Returns array of owners |
| ADM-07 | Suspend owner | Owner's dimes status changed to 'paused' |
| ADM-08 | Activate owner | Owner's dimes status changed to 'active' |
| ADM-09 | Reset password | Password reset successfully |
| ADM-10 | Get audit logs | Returns audit log entries |
| ADM-11 | Filter audit logs | Returns filtered results |

#### Test Commands

```bash
# Get JWT token for admin auth
TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@test.com","password":"password123"}' | jq -r '.token')

# ADM-01: Get stats
curl http://localhost:3000/api/admin/stats \
  -H "Authorization: Bearer $TOKEN"

# ADM-02: List playgrounds
curl http://localhost:3000/api/admin/playgrounds \
  -H "Authorization: Bearer $TOKEN"

# ADM-03: Create playground
curl -X POST http://localhost:3000/api/admin/playgrounds \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"Name": "Test Playground", "type": "social", "maxAgents": 100}'

# ADM-04: Update playground
curl -X PUT http://localhost:3000/api/admin/playgrounds/PG_ID \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"maxAgents": 200}'

# ADM-05: Delete playground
curl -X DELETE http://localhost:3000/api/admin/playgrounds/PG_ID \
  -H "Authorization: Bearer $TOKEN"

# ADM-06: List owners
curl http://localhost:3000/api/admin/owners \
  -H "Authorization: Bearer $TOKEN"

# ADM-07: Suspend owner
curl -X POST http://localhost:3000/api/admin/owners/OWNER_ID/suspend \
  -H "Authorization: Bearer $TOKEN"

# ADM-08: Activate owner
curl -X POST http://localhost:3000/api/admin/owners/OWNER_ID/activate \
  -H "Authorization: Bearer $TOKEN"

# ADM-09: Reset password
curl -X POST http://localhost:3000/api/admin/owners/OWNER_ID/reset-password \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"newPassword": "newpass123"}'

# ADM-10: Get audit logs
curl http://localhost:3000/api/admin/audit \
  -H "Authorization: Bearer $TOKEN"

# ADM-11: Filter audit logs
curl "http://localhost:3000/api/admin/audit?action=create_playground" \
  -H "Authorization: Bearer $TOKEN"
```

### 3.4 FR-MP - Digital Goods Marketplace

| Test ID | Test Case | Expected Result |
|---------|-----------|-----------------|
| MP-01 | Browse goods | Returns goods catalog |
| MP-02 | Publish goods | Goods published, ID returned |
| MP-03 | Get goods details | Returns goods with full details |
| MP-04 | Purchase as owner | Purchase successful, no scope check |
| MP-05 | Get owned goods | Returns list of owned goods |
| MP-06 | Get dime scope | Returns DimeScope with limits |
| MP-07 | Update dime scope | Scope updated successfully |
| MP-08 | Purchase as dime (within scope) | Purchase succeeds |
| MP-09 | Purchase as dime (exceeds scope) | Returns `SCOPE_LIMIT_EXCEEDED` |
| MP-10 | Equip goods | Goods status changed to 'equipped' |
| MP-11 | Configure goods | Per-dime config saved |

#### Test Commands

```bash
# MP-01: Browse goods
curl http://localhost:3000/api/marketplace/goods

# MP-02: Publish goods
curl -X POST http://localhost:3000/api/marketplace/publish \
  -H "Content-Type: application/json" \
  -d '{
    "developerId": "OWNER_ID",
    "type": "skill",
    "name": "Web Search",
    "description": "Search the web",
    "price": 10,
    "pricingType": "one-time",
    "category": "productivity"
  }'

# MP-03: Get goods details
curl http://localhost:3000/api/marketplace/goods/GOODS_ID

# MP-04: Purchase as owner
curl -X POST http://localhost:3000/api/marketplace/goods/GOODS_ID/purchase \
  -H "Content-Type: application/json" \
  -d '{"buyerType": "owner", "dimeId": "DIME_ID", "ownerId": "OWNER_ID"}'

# MP-05: Get owned goods
curl "http://localhost:3000/api/marketplace/my?dimeId=DIME_ID" \
  -H "x-owner-id: OWNER_ID"

# MP-06: Get dime scope
curl http://localhost:3000/api/marketplace/scope/DIME_ID \
  -H "x-owner-id: OWNER_ID"

# MP-07: Update dime scope
curl -X PUT http://localhost:3000/api/marketplace/scope/DIME_ID \
  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
  -d '{"maxSpendPerTransaction": 100, "dailyLimit": 500, "monthlyBudget": 2000}'

# MP-08: Purchase as dime (within scope)
curl -X POST http://localhost:3000/api/marketplace/goods/GOODS_ID/purchase \
  -H "Content-Type: application/json" \
  -d '{"buyerType": "dime", "dimeId": "DIME_ID"}'

# MP-09: Purchase as dime (exceeds scope) - expect failure
curl -X POST http://localhost:3000/api/marketplace/goods/GOODS_ID/purchase \
  -H "Content-Type: application/json" \
  -d '{"buyerType": "dime", "dimeId": "DIME_ID"}'

# MP-10: Equip goods
curl -X POST http://localhost:3000/api/marketplace/equip/DIME_GOODS_ID \
  -H "x-owner-id: OWNER_ID"

# MP-11: Configure goods

  -H "Content-Type: application/json" \
  -H "x-owner-id: OWNER_ID" \
```

## 6. Notes

- Tests require a running server and registered owner with created dimes
- D2D tests require two dimes (from different owners since MVP limits 1 dime per owner)
- Admin routes require Bearer JWT token authentication
- Privacy test verifies owners cannot access their dime's D2D conversations
- Scope validation test verifies dime purchases are constrained by DimeScope
